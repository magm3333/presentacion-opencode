## Slides Como Plan

Cada slide se trata como una unidad planificable:

| Campo | Uso |
|---|---|
| Estado | idea, borrador, anotada, revisada, final |
| Objetivo | Que debe lograr la slide |
| Anotaciones | Correcciones del usuario |
| Accion final | Como se transforma en `slides.md` |

---

## Desglose: Estado

La primera fila indica en que punto esta cada slide:

| Estado | Implica |
|---|---|
| `idea` | Existe la intencion, todavia no se desarrollo |
| `borrador` | Tiene contenido inicial para revisar |
| `anotada` | Tiene comentarios pendientes de aplicar |
| `revisada` | Ya incorpora correcciones |
| `final` | Lista para pasar a `slides.md` |

---

## Desglose: Objetivo

El objetivo define el criterio de aceptacion de la slide.

- Que debe entender el publico.
- Que mensaje no debe perderse.
- Que contenido queda fuera para no saturar.

---

## Desglose: Anotaciones

Las anotaciones capturan correcciones sin romper el flujo.

- Quedan asociadas a una slide concreta.
- Evitan perder decisiones durante la iteracion.
- Alimentan la siguiente version de `slides.md`.

---

## Desglose: Accion Final

La accion final traduce la correccion en cambio concreto.

- Indica como debe quedar la slide.
- Sirve de puente entre `slides-plan.md` y `slides.md`.
- Permite regenerar la presentacion de forma controlada.

## OpenCode: Instalacion

Opcion recomendada por script oficial:

```bash
curl -fsSL https://opencode.ai/install | bash
```

Alternativas:

```bash
npm install -g opencode-ai
brew install anomalyco/tap/opencode
```


## Metodo De Demo: QRGen

![bg right:38% fit](assets/images/qrgen-roadmap.svg)

Una fase = un resultado visible

- **Fase 1:** Estructura base y Auto-deploy.
- **Fase 2:** Logica de generacion QR (MVP).
- **Fase 3:** Descarga y formatos (JPG/PNG).
- **Fase 4:** Editor Visual Basico.
- **Fase 5:** Interaccion Profesional.

---

## Fase 1: Estructura Base y Deploy

![bg right:36% fit](assets/images/qrgen-phase-1.svg)

1. **Bootstrap:** `npm create vite@latest . -- --template vue`
2. **Config:** `base: '/qrgen_demo/'` en `vite.config.js`
3. **Deploy:** GitHub Actions para `gh-pages`.

---

## Fase 2: Generador QR Minimo

![bg right:36% fit](assets/images/qrgen-phase-2.svg)

1. **Libreria:** `npm install qrcode`
2. **Componente:** `src/components/QRGenerator.vue`
3. **Validacion:** `!npm run dev`

---

## Fase 3: Login Estatico SHA256

![bg right:36% fit](assets/images/qrgen-phase-3.svg)

Seguridad local sin Backend:

- **Hash:** `echo -n "pass" | sha256sum`
- **Crypto:** Uso de `crypto.subtle.digest`
- **Sesion:** `localStorage` persistente.

---

## Fase 4: Editor Visual Basico

![bg right:36% fit](assets/images/qrgen-phase-4.svg)

Composición visual sobre fondo:

- Carga de imagen local (Fondo).
- QR flotante sobre la plantilla.
- Lienzo Canvas con dimensiones fijas.

---

## Fase 5: Interaccion Profesional

![bg right:36% fit](assets/images/qrgen-phase-5.svg)

Refinamiento de UX:

- Handles de control (escalar y rotar).
- Interaccion directa con el mouse.
- Cursores dinámicos (`nwse-resize`).

---

## Fase 6: Proyectos Y Exportacion

- Persistencia en JSON (Guardar/Cargar).
- Exportacion a PNG mediante Canvas API.
- Recuperación automática del último trabajo.
